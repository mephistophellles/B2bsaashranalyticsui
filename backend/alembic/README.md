# Миграции Alembic

Из каталога `backend/` (с установленным `PYTHONPATH=.` или виртуальным окружением):

```bash
alembic upgrade head
```

Создать новую ревизию после изменения моделей:

```bash
alembic revision --autogenerate -m "описание"
```

**Уже существующая БД без истории Alembic:** если таблицы созданы через `create_all` и совпадают с моделями, можно пометить текущую ревизию без применения SQL:

```bash
alembic stamp 20250328_01
```

После этого дальнейшие `upgrade` накатывают только новые файлы в `versions/`.
